# 求职规划智能体 JobPlanning Agent — 项目介绍与落地文档

## 一、项目概述

**求职规划智能体** 是一款基于 LangGraph ReAct 范式的 AI 求职规划系统。用户上传简历后，Agent 自主调用 9 个工具（知识库检索、简历解析、JD 解析、人岗匹配评分、分阶段规划生成、联网搜索、长期记忆存取），完成「现状诊断 → 差距分析 → 路径规划 → 执行反馈 → 记忆持久化」的完整闭环。

**一句话定位**：一个真正能跑的、有记忆的、可自主决策的求职规划 AI Agent，而非简单的 RAG 问答机器人。

## 二、技术架构

```
┌─────────────────────────────────────────────────┐
│  前端层：Streamlit Web UI                        │
│  简历上传(PDF/Word)、对话交互、档案管理            │
├─────────────────────────────────────────────────┤
│  Agent 编排层：LangGraph create_react_agent       │
│  SystemPrompt → LLM 推理 → 工具调用 → 回复生成    │
│  循环决策，直到模型认为任务完成                     │
├─────────────────────────────────────────────────┤
│  核心能力层                                       │
│  ┌──────────┬──────────┬──────────┬───────────┐ │
│  │ RAG 知识库│ 简历解析 │ JD 解析  │ 匹配分析  │ │
│  │ (FAISS)  │(Pydantic)│(Pydantic)│  (LLM)   │ │
│  ├──────────┼──────────┼──────────┼───────────┤ │
│  │ 规划生成  │ 联网搜索 │ 长期记忆 │ 记忆查询  │ │
│  │  (LLM)   │ (Tavily) │ (JSON)   │ (CRUD)   │ │
│  └──────────┴──────────┴──────────┴───────────┘ │
├─────────────────────────────────────────────────┤
│  基础模型层                                       │
│  DeepSeek-chat (LLM) + Tavily (搜索)              │
│  BAAI/bge-small-zh-v1.5 (嵌入，可选)              │
└─────────────────────────────────────────────────┘
```

### Agent 编排范式

采用 **ReAct 单 Agent** 模式（Reasoning + Acting），由 LangGraph 的 `create_react_agent` 驱动：

```
用户输入 → LLM 推理 → [需要工具?] → 执行工具 → LLM 再次推理 → 最终回复
              ↑                                    |
              └──────────── 循环 ───────────────────┘
```

工具调用顺序完全由 Agent 自主决定，非预设流程。

## 四、实际落地功能（全部已测试验证）

### 4.1 简历智能解析

**输入**：用户上传 PDF/Word 简历，或直接粘贴文本

**Agent 行为**：自动调用 `resume_analysis` 工具 → LLM 提取结构化画像

**实际落地验证**：
```
输入: "张三 | 本科-计算机 | Python,STM32 | 3年嵌入式开发 | 目标AI工程师"

输出: {
  "name": "张三",
  "education": "本科-计算机",
  "work_years": 3.0,
  "core_skills": ["Python", "STM32"],
  "target_position": "AI工程师",
  ...
}
→ [MEMORY] 已保存用户画像: 张三   # 自动持久化
```

### 4.2 人岗匹配分析

**输入**：用户粘贴目标岗位 JD

**Agent 行为**：调用 `jd_analysis` → `match_analysis` → 返回匹配度评分 + 差距诊断

**实际输出结构**：
- 综合匹配度评分（0-100）
- 用户核心优势列表
- 差距分析（硬技能/经验/学历三级分类）
- 优先级排序的弥补建议

### 4.3 分阶段求职规划

**输入**：用户画像 + 目标岗位 + 差距报告

**Agent 行为**：调用 `plan_generation` → 按周拆解 3 个月计划

**实际输出结构**：
- 按周拆分，每阶段明确核心目标
- 覆盖：技能提升 / 项目补充 / 简历优化 / 投递策略 / 面试准备
- 每周时间分配建议 + 验收标准

### 4.4 联网实时搜索

**场景**：用户问「深圳 AI 工程师薪资」

**Agent 行为**：调用 `internet_search` → Tavily 抓取实时数据 → 整理回复

**实际验证输出**：
```
好的，我查到了深圳AI工程师的最新薪资行情：

## 📊 深圳AI工程师薪资全景
- 应届生：10K-15K
- 1-3年经验：15K-25K
- 3-5年经验：25K-40K
- 5年以上经验：35K-65K·16薪
...
```

### 4.5 知识库 RAG 问答

**输入**：「简历中的项目经历应该怎么写？」

**Agent 行为**：调用 `search_knowledge` → FAISS 检索本地知识库 → 生成回答

**知识库文档**（已内置 3 篇）：
- 简历写作指南.md
- 面试方法论.md
- 嵌入式开发能力模型.md

### 4.6 长期记忆（跨 Session 持久化）

**实现机制**：
```
上传简历 → resume_analysis → 自动保存到 JSON → 下次访问侧边栏显示档案 → 一键加载
```

**实际验证**：
```
阶段1: 张三上传简历 → [MEMORY] 已保存用户画像: 张三 → 存档列表: ['张三']
阶段2: 模拟新Session → 找到档案: True → Agent回复: "您的档案已确认，当前画像如下..."
阶段3: "根据我的背景推荐学习路线" → 回复内容包含 Python/STM32/嵌入式 ✅
```

### 4.7 简历自动感知

**问题**：用户上传简历后直接提问，Agent 看不到简历

**修复方案**：`resume_shared` 标记 + `_ensure_resume_shared()` 自动注入

**效果**：上传后首次提问时，系统自动先把简历发给 Agent 解析，再回答用户问题
